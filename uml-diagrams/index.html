UML Diagrams

Structural and behavioral models illustrating the architecture and interactions of the SafePath system.

Class Diagram

Describes the static structure of the SafePath system — the classes involved, their attributes, methods, and relationships between them.

Component Diagram

Illustrates how the system is split into components — the routing engine, data layers, APIs, and the interfaces connecting them.

Deployment Diagram

Shows the physical deployment of SafePath — servers, client devices, and the infrastructure nodes that host each system component.

Behavioral

Sequence Diagram — Report

Models the interaction flow when a user submits a safety report, tracing the message through the system components over time.

Behavioral

Sequence Diagram — Route

Captures the step-by-step interaction when a user requests a safe route, from input through risk computation to the returned path.

Behavioral

Use Case Diagram

Defines the interactions between users and the SafePath system — the main actors and the use cases they trigger, such as route requests and hazard reporting.
